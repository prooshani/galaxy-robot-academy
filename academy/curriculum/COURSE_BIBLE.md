# Galaxy Robot Academy — Course Bible
## Version 1.0

## Identity

**Course:** Galaxy Robot Academy
**Tagline:** Build. Think. Explore.
**Primary project:** R0-B0 — Mission to Andromeda
**Audience:** Curious children aged approximately 9–12
**Initial cohort:** Parshan and Adryan
**Format:** 12 core sessions over 6 weeks, 2 sessions per week
**Session length:** 75–85 minutes
**Language:** Python
**Main goal:** Teach software-engineering thinking through a continuous robot-and-space project.

Python is the implementation language. The deeper subject is decomposition, prediction, debugging, experimentation, naming, state, decision-making, reuse, testing, and explanation.

## Course promise

Every session must answer:

1. What can R0-B0 do now?
2. Which engineering habit did the students practice?
3. What visible artifact proves progress?

Students do not attend a generic Python class. They join the Academy as junior engineers responsible for bringing R0-B0 to life.

## Teaching principles

- Story before syntax.
- Build before memorize.
- Predict before run.
- Change one thing at a time.
- Debugging is evidence, not failure.
- One primary Python concept per session.
- Creativity is part of the learning target.
- Galaxy Energy is never removed.
- Students explain code aloud.
- Pair support must not become keyboard takeover.
- Short teaching loops are preferred over long lectures.

## Standard session rhythm

1. Incoming Transmission — 5 minutes
2. Engineering Warm-up — 8–10 minutes
3. New Module — 10–15 minutes
4. Guided Live Coding — 15–20 minutes
5. Independent Mission — 15–20 minutes
6. Debugging Game — 8–10 minutes
7. Quick Quiz and Debrief — 5–8 minutes
8. Homework Briefing — 3–5 minutes

## Course progression

| Session | Capability | Python | Engineering mindset |
|---:|---|---|---|
| 1 | Speech Module | `print()`, strings, order | Precise instructions |
| 2 | Memory Core | variables, strings, integers, arithmetic | State and meaningful names |
| 3 | Communication Channel | `input()`, conversion, f-strings | Interactive systems |
| 4 | Decision Core | `if`, `elif`, `else`, comparisons | Rules and branching |
| 5 | Repeat Protocol | `for`, `range()` | Automating repetition |
| 6 | Navigation Scanner | loops + conditions + counters | Combining simple ideas |
| 7 | Cargo Bay | lists | Collections |
| 8 | Skill Modules | functions | Reuse and decomposition |
| 9 | Planetary Database | dictionaries | Structured entities |
| 10 | Event Response | `random`, state changes | Uncertainty |
| 11 | Launch Assembly | integration, refactoring, testing | Systems thinking |
| 12 | Mission to Andromeda | final integration | Presentation and iteration |

## Story arc

### Act I — Awakening
Sessions 1–4: R0-B0 learns to speak, remember, interact, and decide.

### Act II — Preparation
Sessions 5–8: repetition, navigation, inventory, and reusable abilities.

### Act III — Exploration
Sessions 9–10: planet data and unpredictable events.

### Act IV — Launch
Sessions 11–12: integration, testing, presentation, and final mission.

## R0-B0 personality

R0-B0 is curious, brave, slightly humorous, and never patronizing.

- Session 1: quiet and dependent.
- Session 2: forgetful and curious.
- Session 3: friendly and interactive.
- Session 4: cautious and thoughtful.
- Sessions 5–8: increasingly capable and playful.
- Sessions 9–10: confident explorer.
- Sessions 11–12: mission-ready and appreciative.

R0-B0 gives hints, not answers.

## Galaxy Energy economy

A standard session may award approximately 100 GE:

- participation: 15
- guided coding: 15
- independent mission: 25
- debugging: 15
- quiz: 10
- homework: 20
- creative/teamwork bonus: up to 10

Rules:

- no GE removal;
- no humiliating competition;
- no repeated reward farming;
- revision can still earn full required GE;
- bonuses reward effort, explanation, originality, and helpfulness.

## Quiz standard

- 5–8 questions.
- 5–8 minutes.
- Unlimited retries.
- Passing score approximately 70%.
- Immediate gentle feedback.
- Reward granted once.
- Include output prediction, bug identification, concept understanding, and application.
- Avoid trick questions and unexplained terminology.

## Homework standard

- 15–20 minutes.
- Only concepts already taught.
- Visible output.
- One required task and one creative extension.
- Code submission plus one reflection response.
- Manual teacher review.
- Revision and resubmission supported.
- No parent teaching required.

## Review standard

Feedback should:

1. name one thing that worked;
2. identify one specific improvement;
3. give one next action;
4. award GE and badges transparently;
5. set a clear status.

Statuses:

- draft;
- submitted;
- awaiting review;
- needs revision;
- approved;
- excellent.

- mission detail;
- objectives;
- required and bonus tasks;
- quiz;
- homework;
- code and reflection submission;
- submission state;

- GE and badge awarding;

Curriculum is the source of truth. UI code must not invent educational content.

## Completion criteria

A student completes the course by:

- completing the core mission sequence;
- submitting the final project;
- explaining key concepts;
- demonstrating debugging behavior;
- understanding order, state, input, conditions, repetition, collections, and reuse;
- presenting the final mission.
