# Session 04 — Activate the Decision Core
## Teacher Guide

**Duration:** 80–90 minutes
**Upgrade:** Decision Core
**Python:** `if`, `elif`, `else`, comparisons
**Engineering:** programs choose actions from explicit rules

## Learning outcomes

Students can write branches, compare values, use indentation, order rules correctly, and build launch and safety decisions.

## Teaching flow

### 0–5 min — Decision Without Rules

Give R0-B0 different energy and danger situations. Pretend it always says “Launch.”

Ask why this is unsafe.

### 5–15 min — Physical Rule Cards

Use cards:

- IF energy is at least 50 → launch.
- ELSE → recharge.

Students stand in the branch matching each value.

### 15–27 min — First Condition

Use Step 1.

Explain the condition, colon, and indented action.

### 27–42 min — Two and Three Branches

Use Steps 2–3.

Test boundary values such as 49, 50, 79, and 80.

### 42–52 min — Text Decisions

Use password comparison.

Explain `==` versus `=`.

### 52–64 min — Guided Planet Safety

Use danger levels and branch order.

### 64–78 min — Independent Mission

Students build their own safety scanner.

### 78–88 min — Debugging, Quiz, Homework

Test edge values and show online homework.

## Common mistakes

- `=` instead of `==`;
- missing colon;
- missing indentation;
- broad condition before specific condition;
- numeric input not converted.

## Differentiation

Struggling: one `if/else` rule.
Advanced: multiple independent rules or several edge tests. Avoid complex Boolean expressions unless students are ready.

## Badge

**Logic Navigator:** correct branches, correct indentation, meaningful rule order, and explanation of one decision.
