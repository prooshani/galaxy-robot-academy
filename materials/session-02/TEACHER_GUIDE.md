# Session 02 — Install the Memory Core
## Teacher Guide

**Duration:** 75–85 minutes
**Upgrade:** Memory Core
**Python:** variables, strings, integers, arithmetic
**Engineering:** meaningful names and stored state

## Learning outcomes

Students can create variables, distinguish text and numbers, perform simple arithmetic, and build a robot profile.

## Teaching flow

### 0–5 min — Memory Failure

Ask R0-B0’s name, energy, and destination. Pretend it forgets each answer.

**Transition:** “Repeating information is not memory. We need labelled storage.”

### 5–15 min — Labelled Box Activity

Draw boxes labelled:

- `robot_name`
- `energy`
- `destination`

Place values inside. Replace one value and ask what changed.

### 15–27 min — First Variables

Use `LIVE_CODING.md`, Steps 1–2.

Explain:

- variable name;
- assignment;
- stored value.

### 27–39 min — Numbers and Arithmetic

Use energy examples. Predict results.

Emphasize that numbers used for arithmetic should not be quoted.

### 39–51 min — Guided System Profile

Build a five-variable R0-B0 profile.

### 51–66 min — Independent Mission

Use Challenge A. Students create a custom exploration robot.

### 66–76 min — Debugging

Focus on missing quotes, invalid names, wrong names, and text-number confusion.

### 76–84 min — Quiz and Homework

Use quiz and show website homework.

## Common mistakes

- `destination = Mars`
- `robot name = "R0-B0"`
- `energy = "100"` followed by subtraction
- printing a variable name that does not exist

## Differentiation

Struggling: three variables and direct output.
Advanced: two robots or several energy calculations.

## Badge

**Memory Engineer:** clear variables, correct number/text use, one arithmetic result, and explanation of a variable.
