# Session 01 — Wake Up R0-B0
## Teacher Guide

**Duration:** 75–85 minutes
**Upgrade:** Speech Module
**Python:** `print()`, strings, execution order
**Engineering:** precise instructions

## Learning outcomes

Students can run Python, use `print()`, predict order, repair simple syntax errors, and create an eight-line startup sequence.

## Preparation

- Confirm Python runs on both devices.
- Create `mission_01.py`.
- Open the support files.
- Publish the mission, quiz, and homework.
- Prepare GE tracking.

## Teaching flow

### 0–5 min — Incoming Transmission

Read:

> R0-B0 has arrived. Its hardware is ready, but its software is empty. Wake it and teach it to speak.

Ask what a robot should say first.

**Transition:** “Before coding, we need exact instructions.”

### 5–15 min — Human Robot Game

One student follows literal directions to pick up an object. Switch roles.

Ask:
- Were the instructions complete?
- What happens when order is wrong?

State:

> Computers follow what we write, not what we meant.

### 15–25 min — First Python Instruction

Use `LIVE_CODING.md`, Steps 1–2.

Explain only:

- `print` is the instruction;
- parentheses contain the message;
- quotation marks identify text;
- Python follows lines from top to bottom.

### 25–40 min — Guided Startup Sequence

Use Steps 3–4.

For each line:

1. predict;
2. run;
3. change;
4. run again.

Each student contributes at least two system ideas.

### 40–58 min — Independent Mission

Use Challenge A.

Students create eight original messages:

1. name;
2. system;
3. tool;
4. destination;
5. warning;
6. response;
7. funny line;
8. launch-ready ending.

### 58–68 min — Debugging Detectives

Use `DEBUGGING_GAME.md`.

Award GE for explaining repairs.

### 68–78 min — Quiz and Debrief

Use `QUICK_QUIZ.md`.

Ask:
- What does `print()` do?
- Why do quotes matter?
- Why does order matter?
- Which bug did you fix?

### 78–83 min — Homework Briefing

Open the website homework and explain code + reflection submission.

## Common mistakes

- `print(Hello)` → ask how Python knows it is text.
- missing `)` → ask what was opened but not closed.
- `Print(...)` → compare capitalization.
- stale output → save file and confirm correct file runs.

## Differentiation

Struggling: copy-change-run with four lines.
Advanced: serious/silly modes, alien interruption, or emergency sequence. Do not introduce loops yet.

## Badge

**First Contact:** working program, four personalized lines, one bug repair, and an explanation of `print()`.
