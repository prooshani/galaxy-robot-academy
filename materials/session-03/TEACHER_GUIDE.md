# Session 03 — Open the Communication Channel
## Teacher Guide

**Duration:** 75–85 minutes
**Upgrade:** Communication Channel
**Python:** `input()`, `int()`, f-strings
**Engineering:** interactive systems receive and respond

## Learning outcomes

Students can ask questions, store answers, personalize output, convert numeric input, and build an interview console.

## Teaching flow

### 0–5 min — One-Way Radio

Demonstrate that R0-B0 only talks and never listens.

Ask:
- Is this a conversation?
- What must interactive software do?

### 5–13 min — Interview Role-play

One student is R0-B0 and asks questions. The other answers. Record answers on paper.

**Transition:** “Python can pause and wait for an answer.”

### 13–25 min — First Input

Use `LIVE_CODING.md`, Steps 1–2.

Explain that `input()` returns text and waits for the user.

### 25–37 min — Personalized Responses

Use f-strings.

Ask students to predict the final sentence before running.

### 37–49 min — Numeric Conversion

Use age or supplies.

Explain:

> `input()` gives text. `int()` changes suitable text into a whole number.

### 49–59 min — Guided Registration Console

Create name, role, planet, and supply questions.

### 59–72 min — Independent Mission

Students create at least five questions and a summary.

### 72–82 min — Debugging, Quiz, Homework

Use the support files and show website submission.

## Common mistakes

- asking but not storing the answer;
- using the wrong variable;
- missing `f` before a formatted string;
- adding a number to unconverted input.

## Differentiation

Struggling: three questions and direct responses.
Advanced: supply total, future age, secret answer, or custom greeting.

## Badge

**Signal Operator:** working interactive program, stored answers, personalized response, and one correct numeric conversion.
