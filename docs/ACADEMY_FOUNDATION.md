# Galaxy Robot Academy Foundation

## Purpose

Galaxy Robot Academy teaches software engineering thinking through a continuous robot adventure.

Python is the tool. The real subject is engineering mindset.

Students should learn to:

- Give clear instructions
- Break problems into smaller parts
- Debug without panic
- Test ideas quickly
- Build confidence through visible progress
- Help each other think

## Audience

Primary students:

- Parshan, age 9
- Adryan, age 10

The class should feel smart, playful, and meaningful. Avoid long lectures, school-style punishment, and abstract topics without an immediate mission.

## Core Story

Year 2149.

Humanity has built R0-B0, a space exploration robot designed for deep space missions. R0-B0 has hardware, sensors, and a mission, but no software yet.

Two junior engineers are recruited by Mission Control to teach R0-B0 one capability per session until the robot is ready for the Mission to Andromeda.

## Teaching Principles

1. Story before syntax.
2. Build before memorize.
3. Every session unlocks a visible robot capability.
4. Every concept must appear in a mission.
5. Mistakes are treated as debugging clues.
6. No points are removed.
7. Curiosity earns rewards.
8. Sessions should be active, short-loop, and hands-on.

## Class Rhythm

Recommended duration: 70-90 minutes.

```text
10 min  Mission story and warm-up
10 min  Whiteboard thinking
20 min  Live coding together
20 min  Student coding
10 min  Debugging game or mini challenge
5 min   Mission complete, rewards, next teaser
```

## Galaxy Energy

Galaxy Energy, abbreviated as GE, replaces generic XP.

GE rewards progress, effort, creativity, and teamwork.

```text
Attendance                    20 GE
Class mission                 40 GE
Homework mission              50 GE
Creative idea                 20 GE
Helping teammate              15 GE
Bug discovered                10 GE
Teacher bonus                 5-50 GE
```

No GE is ever removed.

## Session Progression

```text
Session 01  R0-B0 can speak                 print()
Session 02  R0-B0 can remember              variables
Session 03  R0-B0 can ask questions         input()
Session 04  R0-B0 can decide                if / else
Session 05  R0-B0 can repeat                loops
Session 06  R0-B0 can count and scan        loops + numbers
Session 07  R0-B0 can carry supplies        lists
Session 08  R0-B0 learns reusable skills    functions
Session 09  R0-B0 explores planets          dictionaries / data
Session 10  R0-B0 meets random events       random
Session 11  R0-B0 prepares launch           project assembly
Session 12  Mission to Andromeda            final project
```

## Homework Philosophy

Homework is always a mission from Mission Control.

Rules:

- Maximum expected time: 20 minutes
- Must include a story wrapper
- Must include a visible output
- Must have one optional bonus challenge
- Must be reviewed personally by the teacher
- GE and badges are awarded after review

## Final Project

The final project is Mission to Andromeda.

Students extend the same robot project across the course until R0-B0 can introduce itself, remember data, ask questions, make decisions, repeat tasks, manage supplies, handle random space events, and complete a final mission.
