# AI Implementation Prompt Template

Every coding prompt for an AI implementation agent should follow this structure for clarity, consistency, predictable scope, and reviewable results.

## Title

Give the task a short, unique name that can be used in plans, commits, and reports.

## Objective

State the single implementation outcome and the expected user or system impact.

## Background

Explain why the work is needed and include relevant product or architecture decisions.

## Existing Context

Point to relevant repository files, existing behavior, interfaces, documentation, and patterns the agent must inspect and reuse.

## Requirements

List the exact behavior, content, interfaces, and quality attributes to implement.

## Constraints

Define architecture boundaries, approved tools, compatibility needs, security requirements, and prohibited changes.

## Expected Files

List the files likely to be created, modified, or deleted while allowing evidence-based adjustments after repository inspection.

## Acceptance Criteria

Define observable and testable conditions that determine whether the implementation is correct.

## Validation

Specify required commands, tests, browser checks, and other evidence the agent must collect.

## Deliverables

List the code, tests, documentation, assets, reports, or other artifacts the agent must produce.

## Out of Scope

Explicitly identify related work the agent must not perform.
