# Definition of Done

A task is ready for review only when every applicable criterion is satisfied:

- The requested scope is implemented completely.
- No unrelated changes are included.
- The build passes, if applicable.
- Lint passes.
- Typecheck passes.
- Applicable automated and manual validation passes.
- All modified and created files are documented in the Task Report.
- Deleted files, if any, are documented in the Task Report.
- Dependencies are documented and justified, or the report confirms that none were added.
- Known risks, limitations, and follow-up needs are documented.
- The final diff has been self-reviewed against the work order and Review Checklist.
- A Task Report has been generated with status `READY FOR ARCHITECT REVIEW`.

Meeting this definition makes a task eligible for architecture review; it does not constitute approval.
