# Engineering History

Every sprint creates a new file in `docs/engineering/history/`. The file records the decisions made during the sprint, the rationale behind those decisions, technical debt introduced, future improvements, and lessons learned.

History entries preserve context that may not be visible in the final code. They should be concise, factual, and linked to relevant tasks, commits, pull requests, or architecture decisions when available.

Use the following template:

```markdown
# Sprint Name

## Decisions

* ...

## Rationale

* ...

## Technical debt

* ...

## Future improvements

* ...

## Lessons learned

* ...
```
